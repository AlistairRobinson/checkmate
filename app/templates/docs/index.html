> Documentation

Everything you need to protect your users from phishing

> Endpoints

Get familiar with our four API endpoints:

> Add

Register for an API key using an email address

> Register

Register a user account for a Checkmate code

> Retrieve

Retrieve the Checkmate code for a user account

> Regen

Generate a new Checkmate code for a user account

> Recommended usage

For the best experience using Checkmate, follow our recommended usage guide:

> Registration

> User register for an account with client

> Client registers user account with Checkmate using their private key and the user's email (see Register)

> Checkmate generates a new code for the user's account

> Client displays code to user whenever they log in and instructs them not to dicslose it

> Authentication

> Client requests a user's account code using their private key and email (see Retrieve)

> Checkmate returns the user's account code

> Client includes code in their communication with the user

> User verifies that the code matches the code they received during registration
