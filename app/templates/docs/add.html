> Add

Register for an API key using an email address

[url]/add

> Usage

Registers a client email and generates an API key needed to use other functionality.

Add can be accessed via a HTTP POST request to [url]/add

The POST request takes one mandatory argument, email, which is the client's email

> Parameters

> email

The client's email in string format

> Example

{email:"a@b.com"}

> Result

If successful, returns a JSON object containing the client's newly registered API key, key, in string format along with a status code of 200.

> Errors

> 400 Bad Request

The email already has a registered API key

OR The email parameter was not included

OR The data or content type of the request was not definied or recognised
