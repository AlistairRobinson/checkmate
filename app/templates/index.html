Here's our solution

We're designing a standardised way to mutually authenticate clients across nearly any communication medium.

Checkmate codes are three words and a four digit PIN which are unique to a user's account with a client, which only the client has direct access to.

Users can request and verify a code to authenticate the client themselves.

> {{apis}}

Registered API clients, all with a unique API key and salt values

> {{accounts}}

Registered user accounts, with no sensitive user data stored

> 180 trillion

Possible user account codes, all generated with natural language processing

Checkmate codes are:

Easy to verify

We encourage users to write down their codes in a secure place to make it quick and easy to verify

Difficult to phish

User's don't have to recall codes often, making them harder for attackers to gain through socially engineer

Easy to teach

Teaching users to look for a clearly defined code is easier, and more reliable, than teaching them to look for typos.

Difficult to fake

With so many combinations, actual codes are hard for scammers to fake.
