Here's our solution

We're designing a standardised way to mutually authenticate clients across nearly any communication medium.

Checkmate codes are three words and a four digit PIN which are unique to a user's account with a client, which only the client has direct access to.

Users can request and verify a code to authenticate the client themselves.

> 180 trillion

Possible user account codes, all generated with natural language processing

Registration

  1. User registers with client
  2. Client registers user account with Checkmate using their private key
  3. Checkmate generates a new code for the account
  4. Client displays code to user whenever they log in

Authentication

  1. Client requests a user's account code using their private key
  2. Checkmate returns a user's account code
  3. Client includes code in their communication with the user
  4. User verifies that the code matches the code they received during registration
